## 1. PIPELINE OVERVIEW (HYBRID MODEL)

1. Ingest headline
2. Detect language (fastText, langid)
3. Translate to English (if needed) using MarianMT / NLLB / DeepL
4. Classify:
   - Topic (BART-MNLI or XLM-R)
   - Sentiment (cardiffnlp/twitter-roberta-sentiment)
   - Event extraction (spaCy + OpenIE)
5. Generate embeddings (MiniLM / LaBSE)
6. Store in vector DB (FAISS, Qdrant)
7. Compute under-representation metrics (URI, Lift, z-score)
8. Serve to frontend via FastAPI + React/Next.js

---

## 2. UNDERREPRESENTATION METRICS

- URI(t) = (S_west(t) - S_global(t)) / (S_global(t) + ε)
- Lift(t) = S_global(t) / (S_west(t) + ε)
- z-score(t) = (X - n*p) / sqrt(n*p*(1-p) + ε)
- Optional: JS-divergence per-topic

---

## 3. TRANSLATION VIABILITY

- Headlines are short → fast translation (<500ms)
- Viable for real-time use with batching and caching
- Use MarianMT, NLLB-200, M2M100, DeepL
- Async mode preferred for frontend smoothness
- Store both original + translated headline

---

## 4. SOURCING HEADLINES

**Primary sources**:
- RSS feeds (fast, easy to parse)
- Scraped front pages (custom parsing)
- Aggregator APIs (NewsAPI, GNews, AYLIEN)
- Social signals (X/Twitter, Reddit, Telegram)

Use a `feeds.json` registry:
{ "source": "aljazeera.com", "region": "MENA", "bias": "center-left", "lang": "en", "rss_url": "https://www.aljazeera.com/xml/rss/all.xml" }

---

## 5. HANDLING BIAS IN RSS

- Use AllSides, MediaBiasFactCheck, Ad Fontes to tag feeds
- Balance by region, language, and bias score
- Track topic coverage by outlet bias (e.g., Fox vs NYT vs Al Jazeera)
- Allow frontend filtering by bias, region, source
- Monitor bias metrics over time
